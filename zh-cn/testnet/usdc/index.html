    鑒權/認證

    請先生成測試網 或 主網API KEY。
    REST接口根URL:

    • 測試網:
      https://api-testnet.bybit.com

    • 主網(請根據您的網絡情況選擇以下任意一個地址使用):
      https://api.bybit.com
      https://api.bytick.com

    所有private接口都需要認證，public 接口不需要認證。

    公共參數

    所有接口均為POST接口

    另外我們提供可選的recvWindow參數（單位是毫秒，默認值為5000），來指定請求在多長時間內有效，同時用來防止重放攻擊。

    註意，timestamp參數必須符合如下規則: serverTime - recvWindow <= timestamp < serverTime + 1000，其中serverTime是Bybit服務器時間，您可以通過服務器時間接口獲取。

    構建請求

    1. 参数为json格式

    2. 使用HMAC_SHA256算法對第1步中拼接的query string簽名，並轉換為16進製字符串，得出sign參數。

    3. 將sign參數加到請求參數中即可；註意GET和POST請求參數格式不同，詳見右邊示例。

    行情接口

    以下市場行情數據接口不需要驗權.

    OrderBook

    請求示例

    curl https://api-testnet.bybit.com/option/usdc/openapi/public/v1/order-book \
-H "Content-Type: application/json" \
-d '{"symbol": "BTC-22OCT21-45000-C"}'

    響應示例

    {
  "retCode":0,
    "retMsg":"SUCCESS",
    "result":[
    {
      "price":"9.40065408",
      "size":"155196.7296",
      "side":"BID"
    },
    {
      "price":"9.13032704",
      "size":"40493.4592",
      "side":"ASK"
    }
  ]
}

    獲取Bybit當前Orderbook信息，買/賣側各25檔.

    HTTP 請求

    POST /{category}/usdc/openapi/public/v1/order-book

    期權業務：category=option, 期貨業務：category=perpetual

    請求參數

    參數 是否必須 類型 說明
    symbol true string 合約名稱

    返回參數

    參數 類型 說明
    price string 價格
    size string 倉位
    side string 買賣方向, 期權ASK or BID， 期貨 Buy or Sell

    cursor string 透傳數據

    心跳包/Ping

    如何發送心跳包

    ws.send("ping");

    響應示例

    {"pong": 1535975085152}
    由於網絡或者程序不穩定,我們強烈建議您使用 ping 心跳包來維持連接. 心跳包建議每30秒發送一次.

    客戶端需要定時發送“ping“給Bybit，Bybit服務器返回heartbeat”pong“來顯示客戶端和服務器之間的連接正常。建議至少每30s發送一次。

    如果客戶端沒有正常收到heartbeat的反饋，則需要檢查網絡連接狀態，並重新連接服務器。

    如Bybit服務器，超過3分鐘沒有收到客戶端的“ping”，則服務器會判定客戶端已斷開鏈接（如實際未斷開，此時Bybit服務端也會主動斷開鏈接）。

    如果服務器判定並斷開鏈接，且客戶處於開啟“斷線保護”的狀態，則客戶所有的Active order也會被自動全部取消。
