    鉴权/认证

    请先生成测试网 或 主网API KEY。
    REST接口根URL:

    • 测试网:
      https://api-testnet.bybit.com

    • 主网(请根据您的网络情况选择以下任意一个地址使用):
      https://api.bybit.com
      https://api.bytick.com

    所有private接口都需要认证，public 接口不需要认证。

    公共参数

    需要签名的接口必须包含以下参数:

    • api_key
    • timestamp - UTC毫秒时间戳
    • sign - 请求参数签名

    另外我们提供可选的recvWindow参数（单位是毫秒，默认值为5000），来指定请求在多长时间内有效，同时用来防止重放攻击。

    注意，timestamp参数必须符合如下规则: serverTime - recvWindow <= timestamp < serverTime + 1000，其中serverTime是Bybit服务器时间，您可以通过服务器时间接口获取。

    构建请求

    拼接参数示例（以查询账号余额接口为例）：

    param_str = "api_key=B2Rou0PLPpGqcU0Vu2&leverage=100&symbol=BTCUSDH21&timestamp=1542434791747"

# api_key=B2Rou0PLPpGqcU0Vu2&
# leverage=100&
# symbol=BTCUSDH21&
# timestamp=1542434791747

    参数按照 字母顺序 排列，然后计算sign。

    1. 所有参数按照字母顺序排序，然后按照query string格式拼接。

    2. 使用HMAC_SHA256算法对第1步中拼接的query string签名，并转换为16进制字符串，得出sign参数。

    GET请求格式（查询账号余额）: 

    GET /spot/v1/order?api_key=q1ksyOX2T0G2SkK8nu&qty=100&recvWindow=10000&side=BUY&symbol=BTCUSDT&timestamp=1623208423972&type=MARKET&sign=b452640c21a2c9eaec30d24a9bce1a9660d1fb9d07ccc0d623a2a4fca0940095 HTTP/1.1
Host: api-testnet.bybit.com

    POST请求格式（创建订单）: 

    POST /spot/v1/order HTTP/1.1
Host: api-testnet.bybit.com
Content-Type: application/x-www-form-unlencoded

api_key:q1ksyOX2T0G2SkK8nu
qty:100
recvWindow:10000
side:BUY
symbol:BTCUSDT
timestamp:1623208423972
type:MARKET
sign:b452640c21a2c9eaec30d24a9bce1a9660d1fb9d07ccc0d623a2a4fca0940095

    3. 将sign参数加到请求参数中即可；注意GET和POST请求参数格式不同，详见右边示例。

    HTTP 请求

    POST /spot/v1/order

    请求参数

    参数 是否必须 类型 说明
    symbol true string 币对名称
    qty true number 交易数量(市价买单即type=Market且side=Buy时qty指的是quote currency；比如BTCUSDT的qty指的是USDT的qty, 而非市价买单时qty指向都是BTC)
    side true string 订单方向, BUY(买入做多)/SELL(卖出做空)
    type true string 订单类型, LIMIT/MARKET/LIMIT_MAKER
    timeInForce false string 执行策略
    price false number 订单价格。type字段为MARKET时，price字段为非必须。type字段为LIMIT或LIMIT_MAKER时，price字段为必须
    orderLinkId false string 特殊订单ID，用户自己生成

    返回参数

    参数 类型 说明
    orderId integer 订单ID
    orderLinkId string 特殊订单ID，用户自己生成
    symbol string 币对名称
    transactTime int 交易时间戳
    price float 订单价格，对于LIMIT 订单必需发送
    origQty float 订单数量
    type string 订单类型。可能出现的值只能为:LIMIT(限价)、MARKET_OF_QUOTE（市价买）、MARKET_OF_BASE（市价卖）
    side string 订单方向, BUY(买入做多)/SELL(卖出做空)
    status string 订单状态。可能出现的值为：NEW(新订单，无成交)、PARTIALLY_FILLED（部分成交）、FILLED（全部成交）、CANCELED（已取消）和REJECTED（订单被拒绝）.
    timeInForce string 执行策略
    accountId long 账户ID
    symbolName string 现货币对名称
    executedQty string 请忽略

    IP频率限制

    Bybit会基于请求IP做频率限制，针对不同的请求方法有以下规则：

    • GET方法:
      • 连续2分钟内每秒不超过50个请求
      • 连续5秒内每秒不超过70个请求
    • POST方法:
      • 连续2分钟内每秒不超过20个请求
      • 连续5秒内每秒不超过50个请求

    账户频率限制

    现货鉴权接口（即需要签名的接口）的限频规则是每秒不超过20次

      • 必须小于一百万 (1000000)

        • 如, 若当前市场价(last price) 为 10314, 那么下单价格的最小值为 1031.5。
        • 伪代码(假设价格是0.5的增量):
          IF price > (last_price * 0.1)
// 发送订单
ELSE
//  不会发送订单，因为价格将不被系统接受
      • 如果已持有仓位，那么价格必须高于强平价格
        • 如, 若多仓的强平价格为 5176.5 那么价格最小为 5177. 在做空的情况下，价格必须低于强平价格。
    • 条件单
